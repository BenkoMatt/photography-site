## 1. Overview

The v3.0-pixieset-gallery release completely redesigned the client gallery page (`gallery.html`) to replicate the aesthetic and interaction patterns of [Pixieset](https://pixieset.com), the industry-standard client gallery platform used by professional photographers. The redesign was informed by directly studying and analyzing a live Pixieset gallery, including downloading and inspecting its compiled CSS and HTML structure.

The main marketing site (`index.html`) was unchanged in this release; all changes targeted the gallery experience.

## 3. Changes Made

### 3.1 gallery.html — Full Redesign

The gallery page was rebuilt from the ground up to match Pixieset's visual language:

- Full-screen cover image with overlaid gallery title and date in white type, framed by an elegant thin border.
- "Enter Gallery" scroll button with an animated down-arrow indicator.
- Minimal fixed navbar that appears on scroll: business name left, gallery title centered, icon buttons right.
- SVG icon buttons in the navbar: favorites (heart), share, slideshow, download.
- Updated password gate with cleaner, more minimal styling.

### 3.2 gallery.js — Rewritten

The gallery JavaScript was rewritten to support the new Pixieset-style interactions (masonry rendering, PhotoSwipe-style lightbox, favorites, slideshow, keyboard navigation, per-passcode cover images).

### 3.3 style.css — Pixieset Styling Added

39 Pixieset-specific CSS classes were added to `style.css`, using a `ps-` prefix namespace to avoid collisions with existing site styles. These cover the navbar, cover hero, masonry grid, photo cards, lightbox, and footer.

### 3.4 Per-Passcode Cover Images

Each demo passcode now supports a custom cover image, allowing different galleries to present different cover photography.
