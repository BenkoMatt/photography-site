# v4.0-cms — Content Management System & Visual Admin Panel

**Version:** v4.0-cms (commit a6e118e) → v4.1-theme-mobile-drag (commit f70a55b) → post-fixes
**Live URL:** https://benkomatt.github.io/photography-site/

---

## 1. Overview

The v4.0-cms release introduced a complete Content Management System (CMS) for the Jenna Lynn Photography website. Instead of editing HTML/CSS/JS files directly, the site owner can now manage all content, theme colors, and mobile layout settings through a visual admin panel in the browser. The CMS server reads and writes a single `content.json` file, generates the static site from that JSON, and publishes changes to GitHub Pages with a single button.

The v4.1 update (commit f70a55b) added the Theme & Colors panel, Mobile Layout panel, and drag-and-drop reordering for all repeatable items. Post-fix commits resolved a phone-preview scrolling issue and a mobile About-section stacking issue.

**Total files in project:** 21 (added 5 CMS files to the v3.0 baseline of 16)

## 2. Architecture

```
┌──────────────┐      ┌─────────────────────┐      ┌──────────────────┐
│  content.json │ ←→  │  Python Server       │ ←→  │  Admin Panel     │
│  (14,404 b)   │      │  (cms/server.py)     │      │  (cms/admin/)    │
└──────────────┘      │  port 5000           │      │  (browser)       │
                      └─────────────────────┘      └──────────────────┘
                               │
                      ┌────────┴────────┐
                      │  Live Preview     │
                      │  localhost:5000/  │
                      └────────┬────────┘
                               │
                      ┌────────┴────────┐
                      │  Static Build     │
                      │  index.html       │
                      │  gallery.html     │
                      └────────┬────────┘
                               │
                      ┌────────┴────────┐
                      │  git push to     │
                      │  GitHub Pages    │
                      └─────────────────┘
```

**Data flow:** content.json ↔ Python server (port 5000) ↔ Admin panel (browser) + Live preview (browser) → Rebuild static HTML → git push to GitHub Pages.

## 3. CMS Server (cms/server.py)

| Property | Value |
|----------|-------|
| File | `cms/server.py` |
| Size | 42,023 b |
| Language | Python (stdlib only — no pip dependencies) |
| Port | 5000 |

### Capabilities

| Feature | Endpoint / Function | Description |
|---------|---------------------|-------------|
| Read content | `GET /api/content` | Returns the full content.json |
| Write content | `POST /api/content` | Writes updated content.json; triggers auto-rebuild |
| Publish | `POST /api/publish` | Runs `git add`, `git commit`, and `git push` to GitHub |
| Static site generator | `build_index()`, `build_gallery()` | Builds `index.html` and `gallery.html` from content.json using Python f-strings |
| Live preview | `localhost:5000/` | Serves the generated site for preview before publishing |
| Theme CSS generator | `build_theme_css()` | Creates dynamic CSS from theme/mobile settings; injected into HTML `<head>` |
| Auto-open admin | startup | Opens the admin panel in the default browser on server start |
